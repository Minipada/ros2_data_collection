    Plugins
    Requirement 1.1 DP-PLU-001
    Can send to AWS Kinesis Data Streams through Fluent Bit

    Requirement 1.2 DP-PLU-002
    Can send to AWS S3 through Fluent Bit

    The plugin must be able to send data to an AWS S3 bucket using Fluent Bit's AWS S3 output plugin.
    Requirement 1.3 DP-PLU-003
    Can save in a file through Fluent Bit

    The plugin must be able to save data to a file using Fluent Bit's file output plugin.
    Requirement 1.4 DP-PLU-004
    Can save files metrics through Fluent Bit

    The plugin must be able to save files metrics (i.e. file size, file permissions, etc.) using Fluent Bit's file metrics output plugin.
    Requirement 1.5 DP-PLU-005
    Can send to HTTP through Fluent Bit

    The plugin must be able to send data to an HTTP server using Fluent Bit's HTTP output plugin.
    Requirement 1.6 DP-PLU-006
    Can send to MinIO through Fluent Bit

    The plugin must be able to send data to a MinIO server using Fluent Bit's MinIO output plugin.
    Requirement 1.7 DP-PLU-007
    Can send to PostgreSQL through Fluent Bit

    The plugin must be able to send data to a PostgreSQL database using Fluent Bit's PostgreSQL output plugin.
    Requirement 1.8 DP-PLU-008
    Can send to Slack through Fluent Bit

    The plugin must be able to send data to a Slack channel using Fluent Bit's Slack output plugin.
    Requirement 1.9 DP-PLU-009
    Can send to Stdout through Fluent Bit

    The plugin must be able to output data to the console using Fluent Bit's Stdout output plugin.
    Requirement 1.10 DP-PLU-010
    Can send to RCL stdout

    The plugin must be able to output data to the ROS 2 console using the RCL stdout output plugin.
    Section 2
    Fluent Bit File
    Requirement 2.1 MP-FLB-FILE-001
    Can write data to a file using the Fluent Bit file plugin

    The file plugin should be able to write data to a file in the specified location. The data should be written in the specified format and appended to the file if it already exists.
    Requirement 2.2 MP-FLB-FILE-002
    Can set the file path

    The file plugin must allow the user to specify the path where the output file will be created. The user should be able to specify an absolute or relative path.
    Requirement 2.3 MP-FLB-FILE-003
    Can set the file name

    The file plugin must allow the user to specify the name of the output file. The user should be able to specify a file name with or without an extension.
    Requirement 2.4 MP-FLB-FILE-004
    Can create parent directories if they are not created before

    If the directory where the file is supposed to be saved does not exist, the plugin must be able to create the directory and any required parent directories automatically. The user should not be required to manually create the directory before starting the plugin.
    Section 3
    Fluent Bit File metrics
    Requirement 3.1 MP-FLB-FILE-M-001
    Can track file metadata

    The plugin must be able to track metadata for all files that are sent to their destinations, including the file type, size, and timestamp.
    Requirement 3.2 MP-FLB-FILE-M-002
    Can delete local files

    The plugin must be able to delete local files that have been sent to all their storage destinations, in order to free up disk space.
    Requirement 3.3 MP-FLB-FILE-M-003
    Uses custom Fluent Bit plugin

    The plugin must use a custom Fluent Bit plugin located in the fluent_bit_plugins package to track file metadata and manage file deletion.
    Requirement 3.4 MP-FLB-FILE-M-004
    Supports multiple storage destinations

    The plugin must support tracking files that are sent to multiple storage destinations, such as AWS S3, MinIO, etc.
    Requirement 3.5 MP-FLB-FILE-M-005
    Stores metadata in PostgreSQL database

    The plugin must store file metadata in a PostgreSQL database, including the file name, status, and the destinations it has been sent to.
    Section 4
    Fluent Bit HTTP
    Requirement 4.1 MP-FLB-HTTP-001
    Supports sending records in JSON format

    The HTTP output plugin should allow the user to specify sending records in JSON format through the 'format' parameter. It must be possible to send multiple headers through the 'header' parameter.
    Requirement 4.2 MP-FLB-HTTP-002
    Supports HTTP Basic Authentication

    The HTTP output plugin should allow the user to send HTTP Basic Authentication credentials through the 'http_user' and 'http_passwd' parameters.
    Requirement 4.3 MP-FLB-HTTP-003
    Supports setting HTTP host and port

    The HTTP output plugin should allow the user to set the IP address or hostname of the target HTTP Server and TCP port through the 'host' and 'port' parameters, respectively.
    Requirement 4.4 MP-FLB-HTTP-004
    Supports setting HTTP URI

    The HTTP output plugin should allow the user to set an optional HTTP URI for the target web server through the 'uri' parameter.
    Requirement 4.5 MP-FLB-HTTP-005
    Supports logging the response payload

    The HTTP output plugin should allow the user to specify whether or not to log the response payload through the 'log_response_payload' parameter.
    Requirement 4.6 MP-FLB-HTTP-006
    Supports different date formats

    The system should allow users to specify different date formats for parsing and formatting dates, such as ISO 8601, RFC 3339, Unix timestamp, and custom formats.
    Section 5
    Fluent Bit AWS Kinesis Data Streams
    Requirement 5.1 MP-FLB-KDS-001
    Ingests logs into Amazon Kinesis Data Streams

    The system should be able to ingest logs into Amazon Kinesis Data Streams using the provided Amazon Kinesis Data Streams output plugin. The plugin should support the configuration of the AWS region, the name of the Kinesis Streams Delivery stream to send the logs to, and an optional IAM role ARN for cross-account access.
    Requirement 5.2 MP-FLB-KDS-002
    Supports custom endpoint for Kinesis API

    The Amazon Kinesis Data Streams output plugin should support the configuration of a custom endpoint for the Kinesis API, in case a user wants to send logs to a custom endpoint instead of the default AWS endpoint.
    Requirement 5.3 MP-FLB-KDS-003
    Allows customization of log record sent to Kinesis

    The Amazon Kinesis Data Streams output plugin should allow customization of the log record that is sent to Kinesis. It should support the configuration of a key name to send only the value of that key to Kinesis, and the addition of a timestamp to the record under a specified key name, using a strftime compliant format string for the timestamp.
    Requirement 5.4 MP-FLB-KDS-004
    Handles network errors gracefully

    The Amazon Kinesis Data Streams output plugin should handle network errors gracefully and provide clear error messages to users, indicating the cause of the error. It should also support immediate retry of failed requests to AWS services once, without affecting the normal Fluent Bit retry mechanism with backoff.
    Section 6
    Fluent Bit MinIO
    Requirement 6.1 MP-FLB-MINIO-001
    Can activate secure access to Minio

    The Minio plugin can provide secure access to the Minio server by supporting HTTPS protocol and allowing the use of access keys and secret access keys for authentication.
    Requirement 6.2 MP-FLB-MINIO-002
    Configurable endpoint for Minio

    The Minio plugin must allow the configuration of the endpoint for the Minio server, including the hostname, port, and scheme.
    Requirement 6.3 MP-FLB-MINIO-003
    Support for access keys and secret access keys

    The Minio plugin must allow the configuration of access keys and secret access keys to provide authentication to the Minio server.
    Requirement 6.4 MP-FLB-MINIO-004
    Support for SSL/TLS encryption

    The Minio plugin must support SSL/TLS encryption to ensure secure communication with the Minio server.
    Requirement 6.5 MP-FLB-MINIO-005
    Configurable bucket creation

    The Minio plugin must allow the configuration of whether a bucket should be created automatically if it does not already exist.
    Requirement 6.6 MP-FLB-MINIO-006
    Configurable bucket name

    The Minio plugin must allow the configuration of the bucket name to which files will be uploaded.
    Requirement 6.7 MP-FLB-MINIO-007
    Configurable upload and source fields

    The Minio plugin must allow the configuration of the fields containing the remote and local paths for file uploads. These fields should be separated by dots.
    Section 7
    PostgreSQL
    Requirement 7.1 MP-FLB-PSQL-001
    Supports PostgreSQL 10 and above

    The plugin should be able to connect and write data to PostgreSQL 10 and above.
    Requirement 7.2 MP-FLB-PSQL-002
    Handles database connection errors gracefully

    The plugin should handle errors when connecting to the PostgreSQL database gracefully and provide clear error messages to users.
    Requirement 7.3 MP-FLB-PSQL-003
    Handles SQL query errors gracefully

    The plugin should handle errors when executing SQL queries gracefully and provide clear error messages to users.
    Requirement 7.4 MP-FLB-PSQL-004
    Supports both async and sync connections

    The plugin should support both async and sync connections to the PostgreSQL database.
    Requirement 7.5 MP-FLB-PSQL-005
    Allows customization of connection pool size

    The plugin should allow users to customize the minimum and maximum number of connections in the connection pool.
    Requirement 7.6 MP-FLB-PSQL-006
    Allows specification of table to store data in

    The plugin should allow users to specify the name of the table in the PostgreSQL database where data should be stored.
    Requirement 7.7 MP-FLB-PSQL-007
    Allows specification of timestamp field

    The plugin should allow users to specify which field in the incoming data contains the timestamp for the record.
    Requirement 7.8 MP-FLB-PSQL-008
    Supports CockroachDB

    The plugin should be able to connect and write data to CockroachDB when the "cockroachdb" option is set to true.
    Requirement 7.9 MP-FLB-PSQL-009
    Can set a timeout for remote connection to PostgreSQL

    The system should be able to set a timeout for remote connection to PostgreSQL and terminate them if no data has been received within the timeout.
    Section 8
    Fluent Bit S3
    Requirement 8.1 MP-FLB-S3-001
    Allows specifying the AWS region of the S3 bucket

    The plugin should allow users to specify the AWS region where the S3 bucket resides.
    Requirement 8.2 MP-FLB-S3-002
    Allows specifying the S3 bucket name

    The plugin should allow users to specify the name of the S3 bucket where data should be stored.
    Requirement 8.3 MP-FLB-S3-003
    Allows specifying the time key in the output record

    The plugin should allow users to specify the name of the time key in the output record. If the time key is disabled, the plugin should not include any time information in the S3 object.
    Requirement 8.4 MP-FLB-S3-004
    Allows specifying the format of the date

    The plugin should allow users to specify the format of the date in the output record. The supported formats should include double, epoch, iso8601, and java_sql_timestamp.
    Requirement 8.5 MP-FLB-S3-005
    Allows specifying the maximum size of files in S3

    The plugin should allow users to specify the maximum size of files that can be uploaded to S3. The maximum size should be 50GB and the minimum size should be 1MB.
    Requirement 8.6 MP-FLB-S3-006
    Allows specifying the size of each 'part' for multipart uploads

    The plugin should allow users to specify the size of each 'part' for multipart uploads. The maximum size should be 50MB.
    Requirement 8.7 MP-FLB-S3-007
    Allows specifying the time interval for completing an upload and creating a new file in S3

    The plugin should allow users to specify the time interval for completing an upload and creating a new file in S3. If this amount of time has elapsed, Fluent Bit will complete an upload and create a new file in S3.
    Requirement 8.8 MP-FLB-S3-008
    Allows specifying the directory to locally buffer data before sending

    The plugin should allow users to specify the directory where data should be buffered before being sent to S3. When multipart uploads are used, data will only be buffered until the upload_chunk_size is reached. S3 will also store metadata about in-progress multipart uploads in this directory.
    Requirement 8.9 MP-FLB-S3-009
    Allows specifying the size limitation for disk usage in S3

    The plugin should allow users to specify the size limitation for disk usage in S3. The amount of S3 buffers in the store_dir should be limited to this size to limit disk usage. Note: Use store_dir_limit_size instead of storage.total_limit_size, which can be used for other plugins, because S3 has its own buffering system.
    Requirement 8.10 MP-FLB-S3-010
    Support for multipart upload

    The plugin should support multipart upload to S3, allowing large files to be uploaded in parts for increased reliability and faster upload times.
    Requirement 8.11 MP-FLB-S3-011
    Can set custom endpoint

    The plugin should support multipart upload to S3, allowing large files to be uploaded in parts for increased reliability and faster upload times.
    Section 9
    Fluent Bit Slack
    Requirement 9.1 MP-FLB-SLACK-001
    Can set the webhook used by Slack

    The plugin should support using the webhook to know where to send the data.
