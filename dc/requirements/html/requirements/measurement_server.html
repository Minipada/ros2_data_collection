    1. Node parameters

         1.1. Can load any number of measurement plugins UID: MS-REQ-001 statement:

    Measurement server can load any number of measurement plugins

         1.2. Can only load existing measurement plugins UID: MS-REQ-002 statement:

    Measurement server throws a custom exception when loading a measurement plugin and stop everything

         1.3. Can load any number of condition plugins UID: MS-REQ-003 statement:

    Measurement server can load any number of condition plugins

         1.4. Can only load existing condition plugins UID: MS-REQ-004 statement:

    Measurement server throws a custom exception when loading a condition plugin and stop everything

         1.5. Can set local base path used by all nodes UID: MS-REQ-005 statement:

    The server must be able to load a path from save_local_base_path parameter

         1.6. Can set local base path used by all nodes and expand with time variable UID: MS-REQ-006 statement:

    Strings as %Y, %M, %D etc. must expand to the current year, month and day. All standard ones should be handled. When encountering one unknown, it should stop the node.

         1.7. Can set local base path used by all nodes and expand environment variables UID: MS-REQ-007 statement:

    Strings starting with $, e.g $HOME must expand as environment variables. When encountering one empty, it should stop the node.

         1.8. Can load custom variables UID: MS-REQ-008 statement:

    The parameter custom_str_params should allow to list custom variables. the subparameter name and one of value_from_file or value must be present. Stop the node otherwise.

         1.9. Can load custom variables from file UID: MS-REQ-009 statement:

    The parameter custom_str_params should allow to list custom variables. Variables can come from a file and stop the node if the file does not exist.

         1.10. Can load custom variables from a fixed string UID: MS-REQ-010 statement:

    The parameter custom_str_params should allow to list custom variables. Variables can come from a fixed string and stop the node if the string is empty.

         1.11. Each measurement can be grouped by setting a group key parameter UID: MS-REQ-011 statement:

    Measurements can be loaded using a group parameter to create a new JSON.

     comment:

    Measurement 1:

    {"a": 1}

    Measurement 2:

    {"b": 2}

    Grouped sample:

    {"a": 1, "b": 2}

         1.13. Can pass a custom JSON schema UID: MS-REQ-013 statement:

    We can define a custom JSON file for our own validation.

    Can apply a filter to keys

         1.15. Can remove a field through a filter UID: MS-REQ-015 statement:

    Apply a filter that remove the key from the sample

         1.16. Match measurement samples to destination UID: MS-REQ-016 statement:

    Can match which sample is sent where using tags

         1.17. Match measurement samples to destination UID: MS-REQ-017 statement:

    Can match which sample is sent where using tags

         1.18. Can collect also when the node starts UID: MS-REQ-018 statement:

    ROS timers by default do an action after the time of the wall timer has elapsed. We need an option to collect a sample when starting directly.

         1.19. Can collect also when the node starts UID: MS-REQ-019 statement:

    ROS timers by default do an action after the time of the wall timer has elapsed. We need an option to collect a sample when starting directly.

         1.20. Can collect when the node starts a certain amount of samples UID: MS-REQ-020 statement:

    Stop collecting after a certain amount of samples has been reached

         1.21. Publisher is deleted when max samples reached UID: MS-REQ-021 statement:

    Stop the publisher, even the loop when the maximum amount of samples has been reached.

         1.22. Can include measurement name in sample UID: MS-REQ-022 statement:

    Each sample can have (or not) the measurement name included as key/value

         1.23. Can include measurement plugin name in sample UID: MS-REQ-023 statement:

    Each sample can have (or not) the measurement plugin name included as key/value

         1.24. Can pass remote keys UID: MS-REQ-024 statement:

    Can precise the remote key, based on remote destination

         1.25. Can apply a prefix to the remote key UID: MS-REQ-025 statement:

    Can pass a prefix to the remote key, to add a string to the path

         1.26. Can send samples if all conditions are respected (from a list of condition) UID: MS-REQ-026 statement:

    Can pass a list of conditions. All need to be all respected to send data sample

         1.27. Can send samples if any conditions is respected (from a list of condition) UID: MS-REQ-027 statement:

    Can pass a list of conditions. One of them need to be respected to send data sample

         1.28. Can send samples if no conditions are not respected (from a list of condition) UID: MS-REQ-028 statement:

    Can pass a list of conditions. None need to be respected to send data sample

         1.29. Can send a maximum number of samples when conditions are met UID: MS-REQ-029 statement:

    Sometimes, it is useful to send data when conditions are respected and only a certain amount of time

         1.30. Stop all nodes if one fails UID: MS-REQ-030 statement:

    Include a bond connection for the lifecycle manager to ensure that after a server transitions up, it also remains active. If a server crashes, it lets the lifecycle manager know and transition down the system to prevent a critical failure.
