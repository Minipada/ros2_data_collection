    Section 2
    Camera plugin
    Requirement 2.1 MP-CAM-001
    Can set camera name

    Each camera can be identified by a unique alias
    Requirement 2.2 MP-CAM-002
    Can set camera topic

    Set from where which ROS topic the camera data comes from
    Requirement 2.3 MP-CAM-003
    Can apply transformations to an image

    Before inspecting the data, it can be transformed by passing a list of transformations to apply. Each transformation has a name, type and parameters.
    Requirement 2.4 MP-CAM-004
    Can apply rotation transformation to an image

    Able to rotate an image to 90, 180 degrees
    Requirement 2.5 MP-CAM-005
    Can save raw image

    Raw image can be saved as a file by setting a parameter.
    Requirement 2.6 MP-CAM-006
    Can save transformed image

    Transformed image can be saved as a file by setting a parameter.
    Requirement 2.7 MP-CAM-007
    Can save inspected image

    Inspected image can be saved as a file by setting a parameter.
    Requirement 2.8 MP-CAM-008
    Can set path for where to save raw image

    Can set a path where to save camera raw images. Able to expand times:

    camera/raw/%Y-%m-%dT%H:%M:%S

    becomes for example camera/raw/2023-03-03T09:01:23
    Requirement 2.9 MP-CAM-009
    Can set path for where to save transformed image.

    Able to set a path where to save camera transformed images.

    Can expand times:

    camera/transformed/%Y-%m-%dT%H:%M:%S

    becomes for example camera/transformed/2023-03-03T09:01:23

    Requirement 2.10 MP-CAM-010
    Can set path for where to save inspected image

    Able to set a path where to save camera inspected images.

    Can expand times:

    camera/inspected/%Y-%m-%dT%H:%M:%S

    becomes for example camera/inspected/2023-03-03T09:01:23

    Requirement 2.11 MP-CAM-011
    Can set path for where to save inspected image

    Able to set a path where to save camera inspected images.

    Can expand times:

    camera/inspected/%Y-%m-%dT%H:%M:%S

    becomes for example camera/inspected/2023-03-03T09:01:23

    Requirement 2.12 MP-CAM-012
    Can apply translation image transformation

    Image can be translated by shifting its x and y

    See OpenCV documentation

    Requirement 2.13 MP-CAM-013
    Can apply scale image transformation

    Image can be scaled to change its dimension

    See OpenCV documentation

    Requirement 2.14 MP-CAM-014
    Can apply flipping image transformation

    Image can be flipped horizontally or vertically

    See OpenCV documentation

    Requirement 2.15 MP-CAM-015
    Can apply image arithmetic image transformation

    Image can be modified with image arithmetic, for example to make it lighter or darker

    See OpenCV documentation

    Requirement 2.16 MP-CAM-016
    Can apply masking image transformation

    Image can be transformed by using masking to extract only a certain part of an image.

    See OpenCV documentation

    Requirement 2.17 MP-CAM-017
    Can apply affine image transformation

    Image can be rotated with affine transformation

    See OpenCV documentation

    Section 3
    Command velocity plugin
    Requirement 3.1 MP-CMV-001
    Get command velocity sent by the robot

    Linear, angular and computed command velocity need to be collected.
    Requirement 3.2 MP-CMV-002
    Can set topic where command velocity is published

    Command velocity can come from different topic.
    Section 4
    CPU plugin
    Requirement 4.1 MP-CPU-001
    Get average cpu

    Get a percentage of CPU used. Must be between 0 and 100.
    Requirement 4.2 MP-CPU-002
    Get cpu per processes

    Get CPU used by processes. 5 by default and 5.0% cpu used minimum
    Requirement 4.3 MP-CPU-003
    Can set maximum processes to store in sample

    A ROS parameter must be defined for the number of processes matching the minimum cpu usage
    Requirement 4.4 MP-CPU-004
    Can set cpu minimum to store its information

    A ROS parameter must be defined for the minimum CPU usage to store in sample.
    Section 5
    Distance traveled plugin
    Requirement 5.1 MP-DST-001
    Can get distance traveled in meters

    A measurement plugin can send how much the robot moved in meters in a certain amount of time. Still, send 0 if no movement.
    Section 6
    IP Camera plugin
    Requirement 6.1 MP-IPC-001
    Can connect to IP cameras

    The plugin should be able to connect to IP cameras and retrieve video streams for processing.
    Requirement 6.2 MP-IPC-002
    Can configure camera parameters

    The plugin should allow users to configure camera parameters such as resolution, frame rate, compression format, and authentication credentials.
    Requirement 6.3 MP-IPC-003
    Can support multiple camera brands and models

    The plugin should be able to connect to IP cameras from multiple brands and models, and should support different protocols such as RTSP, HTTP, and ONVIF.
    Requirement 6.4 MP-IPC-004
    Can handle network interruptions

    The plugin should be able to handle network interruptions and automatically reconnect to the camera when the network connection is restored.
    Requirement 6.5 MP-IPC-005
    Can handle camera disconnections

    The plugin should be able to handle camera disconnections and automatically reconnect to the camera when it becomes available again.
    Requirement 6.6 MP-IPC-006
    Can provide video stream metadata

    The plugin should be able to extract metadata from the video stream such as resolution, frame rate, duration, and timestamps.
    Requirement 6.7 MP-IPC-007
    Supports video data

    The IP camera input plugin should collect video data
    Requirement 6.8 MP-IPC-008
    Supports audio data

    The IP camera input plugin should collect audio data
    Requirement 6.9 MP-IPC-009
    Supports video resolution configuration

    The IP camera input plugin should support configuration of video resolution to adjust the quality of the video stream.
    Requirement 6.10 MP-IPC-010
    Supports audio bitrate configuration

    The IP camera input plugin should support configuration of audio bitrate to adjust the quality of the audio stream.
    Requirement 6.11 MP-IPC-011
    Supports video segmentations

    The IP camera input plugin should support segmenting the video stream into smaller chunks to reduce the size of each file and facilitate data processing.
    Requirement 6.12 MP-IPC-012
    Supports video frame rate configuration

    The IP camera input plugin should support configuration of video frame rate to adjust the smoothness of the video stream.
    Requirement 6.13 MP-IPC-013
    Supports authentication for accessing the IP camera

    The IP camera input plugin should support authentication mechanisms to secure the connection to the IP camera and prevent unauthorized access.
    Section 7
    Map plugin
    Requirement 7.1 MP-MAP-001
    Can collect ROS 2 Map data

    The system should be able to collect ROS 2 Map data generated by the nav2 map_saver tool.
    Requirement 7.2 MP-MAP-002
    Supports multiple file formats

    The plugin should support multiple file formats for the collected map data, including PNG, YAML, and PGM.
    Requirement 7.3 MP-MAP-003
    Set maximum timeout to save image

    The plugin should support multiple file formats for the collected map data, including PNG, YAML, and PGM.
    Requirement 7.4 MP-MAP-004
    Timeout for save map

    The system should allow the user to set a timeout for nav2 map_saver plugin, which controls the duration for which the plugin will wait for a response before timing out and returning an error. This is necessary to prevent the plugin from waiting indefinitely and blocking other operations in the system.
    Requirement 7.5 MP-MAP-005
    Control path to save map data

    The system should allow the user to specify the path where the map data generated by the nav2 map_saver plugin is saved.
    Section 8
    Memory plugin
    Requirement 8.1 MP-MEM-001
    Can collect memory usage data

    The system should be able to collect data on memory usage in order to monitor and analyze the performance of the system.
    Section 9
    Network plugin
    Requirement 9.1 MP-NET-001
    Network plugin

    The system should include a network plugin that can check the accessibility of a remote address and port, and report the result. The plugin should allow the configuration of the remote address and port to be checked. The plugin should support both TCP and UDP protocols.
    Requirement 9.2 MP-NET-02
    Network Plugin Timeout

    The network plugin must have a configurable timeout parameter for checking remote address and port accessibility. The timeout value must be at least 0.1 second and configurable by the user.
    Section 10
    OS plugin
    Requirement 10.1 MP-OSP-001
    Can collect OS name

    The plugin must be able to identify the name of the operating system it is running on.
    Requirement 10.2 MP-OSP-002
    Can collect kernel version and name

    The plugin must be able to identify the version and name of the kernel used by the operating system.
    Requirement 10.3 MP-OSP-003
    Can collect available memory

    The plugin must be able to retrieve information on the amount of available memory on the system.
    Requirement 10.4 MP-OSP-004
    Can collect CPU count

    The plugin must be able to identify the number of CPUs available on the system.
    Section 11
    Permissions plugin
    Requirement 11.1 MP-PER-001
    Check file/directory permissions

    The system should have a plugin that can check the permissions of a given file or directory. The plugin should be able to determine if the current user has read, write, or execute permissions on the file/directory, and return an appropriate response.
    Section 12
    Position plugin
    Requirement 12.1 MP-POS-001
    Collect robot position

    The system shall include a plugin that can collect the position of the robot in the environment, using sensors such as GPS or odometry data.
    Section 13
    Speed plugin
    Requirement 13.1 MP-SPE-001
    Can collect speed data

    The plugin should be able to collect speed data from a robot running ROS 2 and provide the data in appropriate units (e.g., meters per second).
    Section 14
    Storage plugin
    Requirement 14.1 MS-STO-001
    Can determine available disk space

    The plugin must be able to retrieve information about the amount of free disk space in a specified directory.
    Section 15
    String stamped plugin
    Requirement 15.1 MP-STR-001
    Can collect generic data from a topic publishing a StringStamped message

    The plugin should be able to subscribe to a ROS 2 topic and receive a StringStamped message containing generic data.
    Section 16
    TCP Health plugin
    Requirement 16.1 MP-TCP-001
    Can connect to a TCP server

    The plugin should be able to connect to a specified TCP server to collect its status.
    Requirement 16.2 MP-TCP-002
    Can collect TCP server status

    The plugin should be able to collect the status of the connected TCP server, including information about whether the server is accepting connections, the number of active connections, and other relevant details.
    Requirement 16.3 MP-TCP-003
    Can set connection timeout

    The plugin should allow the user to set a connection timeout value, after which the plugin should terminate the connection attempt if it has not successfully connected to the server.
    Section 17
    Uptime plugin
    Requirement 17.1 MP-UPT-001
    Can collect OS uptime

    The OS plugin should be able to retrieve the time duration since the system was last booted up.
